# Technical Handover

This project is a reusable static-site template with a Node/Express backend for AI chat, lead capture, admin dashboard, email notifications and PostgreSQL storage. The current client implementation is Elite Dentistry.

## Quick Start

```bash
npm install
npm start
```

Local site:

```txt
http://localhost:3000
```

Key runtime endpoints:

```txt
/api/client-config       Public client config JSON
/api/client-theme.css    CSS variables generated from client config
/api/chat                AI chatbot endpoint
/api/send-email          Contact form email + database save
/api/callbacks           Admin lead list API
/admin/callbacks.html    Lead dashboard
/health                  Health check
```

## Main Architecture

```txt
server.js                Express app, static files, route mounting, DB init
config/client.json       Active per-client business, brand and chatbot config
config/system_prompt.txt AI chatbot business/personality/safety prompt
lib/clientConfig.js      Loads client.json, applies .env overrides, exposes theme CSS
routes/ai.js             Chatbot API, lead capture tools, optional web search
routes/email.js          Contact form endpoint, Mailgun emails, Postgres save
routes/callbacks.js      Admin login, lead listing, stats and status updates
db/schema.sql            callback_requests and search_queries schema
public/                  Static website, CSS, JS, images and admin page
```

## Branding And CSS

Main CSS:

```txt
public/css/styles.css
```

Mobile CSS:

```txt
public/css/mobile.css
```

Current mobile styling is intentionally kept inside `styles.css`; `mobile.css` only remains for compatibility.

Dynamic brand variables are generated by:

```txt
lib/clientConfig.js
/api/client-theme.css
```

The public pages include:

```html
<link rel="stylesheet" href="/css/styles.css">
<link rel="stylesheet" href="/api/client-theme.css">
```

Brand colours, fonts and radii are defined in `config/client.json` under:

```json
"design": {
  "colors": {},
  "fonts": {},
  "radius": {}
}
```

These become CSS variables such as:

```css
--color-primary
--color-accent
--color-bg-body
--color-bg-surface
--color-text-main
--font-heading
--font-body
```

`.env` can override these using keys such as:

```txt
COLOR_PRIMARY
COLOR_ACCENT
COLOR_BG_BODY
FONT_HEADING
FONT_BODY
```

## Chatbot

Frontend:

```txt
public/js/aiChat.js
```

Backend:

```txt
routes/ai.js
config/system_prompt.txt
config/client.json
```

The chatbot:

- Loads public business/config data from `/api/client-config`.
- Applies client colours and text.
- Uses `config/system_prompt.txt` as the core business prompt.
- Can capture callback/contact leads into PostgreSQL.
- Can offer chat transcript/download lead capture if enabled.
- Can optionally use Brave web search if enabled.

Main chat config in `config/client.json`:

```json
"chat": {
  "title": "Chat with Elite Dentistry",
  "welcome": "...",
  "inputPlaceholder": "...",
  "suggestedQuestions": []
}
```

Feature flags:

```json
"features": {
  "callbacks": true,
  "downloadLeads": true,
  "bookings": true,
  "webSearch": false
}
```

## Lead Capture

Contact forms use:

```txt
public/js/formHandler.js
routes/email.js
```

Forms should use one of these IDs:

```html
<form id="contactForm">
```

or:

```html
<form id="quoteForm">
```

Expected fields:

```txt
name      required
email     required
message   required
phone     optional
service   optional
```

Submissions are:

1. Validated in the browser.
2. Sent to `/api/send-email`.
3. Saved into `callback_requests`.
4. Emailed to the admin via Mailgun.
5. Auto-replied to the user if email is present.

## Email

Mailgun config:

```txt
MAILGUN_API_KEY
MAILGUN_DOMAIN
MAILGUN_REGION
EMAIL_FROM
RECIPIENT_EMAIL
EMAIL_TO
REPLY_TO_EMAIL
```

`RECIPIENT_EMAIL` is the admin destination used by the app. `EMAIL_TO` is supported as a compatibility alias through `lib/clientConfig.js`.

Email subjects/signoff can be configured in `config/client.json`:

```json
"email": {
  "contactSubjectPrefix": "New Website Enquiry",
  "userContactSubject": "Thanks for contacting Elite Dentistry",
  "brandSignoff": "The Elite Dentistry team"
}
```

## Environment Variables

`.env` is the local/runtime source. `.env.example` mirrors the key set without secrets.

Coolify should receive equivalent values through its environment variable UI.

Important groups:

```txt
PORT
NODE_ENV
SITE_URL
SITE_NAME
SITE_DESCRIPTION
CLIENT_VERTICAL
PUBLIC_PHONE
PUBLIC_EMAIL
BOOKING_URL
MAILGUN_*
EMAIL_*
DATABASE_URL
ADMIN_DASHBOARD_PASSWORD
OPENROUTER_*
ENABLE_*
CHAT_*
COLOR_*
FONT_*
```

Rule of thumb:

- Use `config/client.json` for structured content, services, brand design and chat defaults.
- Use `.env` or Coolify variables for secrets and deployment-specific overrides.
- Do not put API keys or database credentials into JSON config.

## Rebuild Workflow For A New Client

1. Copy `config/client.example.json` into `config/client.json`.
2. Update business identity, services, offer, email, contact and booking values.
3. Update `config/system_prompt.txt` for the client’s vertical, rules and lead flow.
4. Add logo and imagery under `public/images/<client>/`.
5. Update logo paths in `config/client.json`.
6. Rework static pages under `public/`.
7. Update CSS tokens in `config/client.json`; edit `public/css/styles.css` only for layout/system changes.
8. Update `.env` or Coolify variables for deployment URL, email, database, API keys and overrides.
9. Run:

```bash
npm start
```

10. Check:

```txt
/api/client-config
/api/client-theme.css
/health
/admin/callbacks.html
```

11. Submit a test form and check:

```txt
Postgres lead row
Admin email
User auto-reply
Dashboard visibility
```
